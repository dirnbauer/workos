# TYPO3 Security Audit

- Date: 2026-04-22 15:15 Europe/Vienna
- Requested rule: `.cursor/rules/typo3-security.mdc` (not present in this workspace)
- Method: controller/service review, targeted tests, official TYPO3 auth/session documentation

## Result

Clean after fixes.

## Findings addressed

1. Team admin actions only required active membership.
2. Account-center factor/session actions were not bound to the current WorkOS user.
3. Frontend auth flows lacked request-token validation.
4. Credential flows reflected unsanitized `returnTo` targets.

## Actions taken

- Restricted Team admin actions to WorkOS `admin` / `owner` memberships.
- Added factor/session ownership checks before destructive WorkOS API calls.
- Added request-token validation to all frontend auth POST flows.
- Sanitized `returnTo` consistently before TYPO3 session creation.
